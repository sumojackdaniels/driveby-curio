# DriveByCurio — Architecture

## What is DriveByCurio?

A walking tour platform. Users pick a tour from a catalog, start walking, and the app narrates the place around them — waypoint-triggered by GPS. Pre-authored tours ship bundled with ElevenLabs-narrated audio; user-created tours can be recorded on-device.

## App Category

iPhone-only SwiftUI app. No CarPlay in the current build (the CarPlay prototype lives under `archive/carplay-frontend/` for reference but is not included in the Xcode project).

## Architecture

### Single-scene iPhone app

- `DriveByCurioApp` (`@main`) wires `AppState.shared` as an `@Observable` and injects stores into the SwiftUI environment.
- `ContentView` → `WalkingTourBrowserView` is the root.
- `PhoneSceneDelegate` is the only `UISceneDelegate`.

### Data flow

1. `LocationService` (core-swift) streams location + heading at 1 Hz while the app is foregrounded.
2. `WalkingTourPlayer` holds the active tour, computes distance to the next waypoint, and fires waypoint-triggered narration when the user enters a trigger circle.
3. `WalkingTourStore` loads bundled authored tours (`AuthoredWalkingTours`) and any user-created tours persisted via `TourStorageService`.
4. `AudioRecorderService` handles on-device audio recording for the tour-creation flow.

### Dependencies

- **core-swift** (SPM) — `LocationService`, `HeadingCalculator`, `APIClient`. The apps in this organization share this package; walking-tour-specific location work could eventually migrate into it.
- `MapKit.framework` — waypoint map editor, route previews
- `AVFoundation.framework` — playback + recording

### Archive

- `archive/carplay-frontend/` — CarPlay prototype: scene delegate, POI refresh controller, driving-era `Tour`/`TourPlayer`/`TourService`/`TourCatalogStore`, topic/POI stores, `TopicsEditorView`/`TourBrowserView`, CarPlay entitlement, and the original tests. Not compiled.
- `archive/backend/` — Express/TypeScript Cloud Run service for the driving prototype (`POST /nearby`, `GET /tours`, `GET /tours/:id/audio/:file`). Still deployed but unused by the app — see the walking pivot notes for undeploy candidates.
- `archive/specs/` — `CARPLAY-CONSTRAINTS.md`, `MVP-spec.md`, `CURATED-TOURS-MVP.md`, the old driving-focused `PRODUCT.md`, and the CarPlay Developer Guide reference PDF.
- `archive/docs/TESTING.md` — Connecticut Avenue curated-tour test plan.
