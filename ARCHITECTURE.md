# DriveByCurio — Architecture

## What is DriveByCurio?

A walking tour platform. Users pick a tour from a catalog, start walking, and the app narrates the place around them — stop-triggered by GPS. Pre-authored tours ship bundled with ElevenLabs-narrated audio; user-created tours can be recorded on-device.

## App Category

iPhone-only SwiftUI app. No CarPlay in the current build (the CarPlay prototype lives under `archive/carplay-frontend/` for reference but is not included in the Xcode project).

## Architecture

### Single-scene iPhone app

- `DriveByCurioApp` (`@main`) wires `AppState.shared` as an `@Observable` and injects stores into the SwiftUI environment.
- `ContentView` → `WalkingTourBrowserView` is the root.
- `PhoneSceneDelegate` is the only `UISceneDelegate`.

### Data flow

1. `LocationService` (core-swift) streams location + heading at 1 Hz while the app is foregrounded.
2. `WalkingTourPlayer` holds the active tour, computes distance to the next stop, and fires stop-triggered narration when the user enters a trigger circle.
3. `WalkingTourStore` loads bundled authored tours (`AuthoredWalkingTours`) and any user-created tours persisted via `TourStorageService`.
4. `AudioRecorderService` handles on-device audio recording for the tour-creation flow.

### Data model (v2)

Since PR #7 the tour model is stop/segment/path-based:

- `WalkingTour` — top-level tour with `stops`, `paths`, `author`, `coverImageName`, etc.
- `TourStop` — a physical location (lat/lng, trigger radius, address) that contains one or more `segments`.
- `TourSegment` — a single piece of playable content (`narration`, `interview`, `poem`, `photo`, `fieldRecording`, `note`) with its own audio file and duration.
- `TourPath` — precomputed walking/biking times + distance between two consecutive stops.
- `TourAuthor` — name, role, bio, initials for the consumption UI.

`WalkingTour` has a custom `Codable` conformance that decodes the legacy flat-`waypoints` JSON format into the new `stops`/`segments`/`paths` shape, so older on-disk user tours keep working. `WalkingTour.computePaths(from:)` is the single source of truth for deriving paths from GPS coordinates — both the decoder and the tour-creation map editor call it.

### Dependencies

- **core-swift** (SPM) — `LocationService`, `HeadingCalculator`, `APIClient`. Shared with other apps in this org.
- `MapKit.framework` — stop map editor, route previews, `MKMapSnapshotter` for feed-card map inlays.
- `AVFoundation.framework` — playback + recording.

### Tests

`DriveByCurioTests` is an XCTest bundle wired in via `project.yml`. It currently covers pure model logic: `coverQuote`, `computePaths` edge cases, and decoding the legacy flat-waypoint JSON format. Run with:

```
xcodebuild test -project DriveByCurio.xcodeproj -scheme DriveByCurio \
  -destination 'platform=iOS Simulator,name=iPhone 17' \
  -derivedDataPath /tmp/curio-build
```

See [TECHNOLOGIES.md](TECHNOLOGIES.md) for lessons captured during tour-consumption UI work (SwiftUI gotchas, MapKit snapshotting, layout pitfalls, etc.).
